Что произошло: 
после создания проекта, ввода 30 тест кейсов, создания спринта и тест рана, добавления тестов в тест ран, прогона тестов и ввода результатов (не закрывая тест ран) выходим на главную доску со списком проектов, в настройках проекта пытаемся дать доступ к проекту другому пользователю (происходит обрыв с сервером?, т.к. в списке Access users нет ни одного пользователя), протыкиваем другие вкладки - Project tags, Statuses, Custom fields, Shared steps, они пустые не содержат данных).
нажимаем Go back - проект удаляется из списка проектов.
Сессия учетной записи длится 7 часов, не выходим из учетной записи, можно сворачивать браузер, переключаться на другие вкладки. При этом обрыва соединения не было, заново логиниться не приходилось.


Шаги:

Вводим логин и пароль на сайте everyqa.io


1. Нажимаем Add Project

2. В графе Name вводим имя проекта "Calculatore", в графе project type выбираем "Desktop"

3. Нажимаем кнопку Save

4. Открываем проект "Calculatore"

5. В левой панеле переходим на вкладку "Test cases"

6. В поле для ввода задаем имя репозиторию кейсов, в графе Name вводим "Test cases calculatore"

7. Открывается станица для ввода тест кейсов

8. Нажимаем кнопку Add Test Case

9. Вводим имя тест кейса "Операция сложения - 1", нажимаем "галочку" 

10. Заходим в тест кейс "Операция сложения - 1", открывается страница с содержимым тест кейса, нажимаем "карандаш" для редактирования тест кейса и введения шагов.

11. Выставляем значения Section - calculatore, estimate - 1, priority - medium, choose tags - functional, в поле Preconditions пишем "Запускаем программу", в разделе Steps нажимаем кнопку "Add new step" (заполняем шаги и ожидаемый результат).

12. нажимаем вверху кнопку Save

13. нажимаем кнопку Go back

14. Далее вводим тест кейсы и заполняем их (пока количество не станет равно 30 шт.)

15. В левой панели переходим на вкладку Sprints

16. Ничего не меняем, нажимаем кнопку add sprint

17. Переходим в Sprint и нажимаем Add test run

18.В test run добавляем через кнопку "Add test" Test cases (все 30 шт.)

19. Проводим test run, во все тесты со статусом failed в результатах добавляем изображения в формате .png, в notes заполняем фактический результат (на русском языке)

20. По завершении test run не закрываем его, нажимаем кнопку Go back
21. Нажимаем на логотип EveryQA и попадаем на Main Dashboard

22. в списке проектов выбираем "Calculatore", нажимаем на кнопку "настройки" ("шестеренка")

23. Справа во вкладке Users access даем права доступа одному из пользователей в списке, нажимаем кнопку Save

24. Нажимаем Go back, затем на логотип EveryQA 

25. Обновляем страницу
